Interface

Four commands.
Ninety percent of your day.

The advanced surface is there when you need it. Hidden until then.

01
engx run <project>
Validate, start, wait, and confirm — one command. Tells you what failed, where, why, and exactly what to type next.
02
engx ps [project]
Project-level status. Not service IDs and state machines — outcomes. Running or not, and the reason if not.
03
engx logs <service>
Tail live logs for any service. Every state change, start, stop, crash, and recovery is an event in the stream.
04
engx doctor
Full platform diagnosis. Health, ports, permissions, database integrity, binary versions. Tells you what to fix and how.

Platform

Built for systems
with moving parts.

⚡
Process control
Start, stop, restart with backoff. Services move to bounded maintenance on repeated failure — no infinite loops.
🔍
Policy governance
Pre-execution gate (V-rules) and runtime findings (G-rules). Blocks unsafe starts. Surfaces failure patterns automatically.
📡
Structured events
Every state change is a typed event with trace ID, span ID, component, and outcome. Full history. Live SSE stream.
🧠
AI insights
Sentinel correlates crash clusters, deploy timing, and dependency risk. Optional AI narrative on explicit request only.
🔄
Automation engine
Trigger workflows on platform events or cron schedules. All executions logged with trace IDs — manual and automated unified.
🔒
Local-first
Runs entirely on 127.0.0.1. No cloud dependency, no telemetry, no external calls during normal operation.
